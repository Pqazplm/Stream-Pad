## 📌 Описание проекта

Stream Pad Engine — это программное обеспечение для управления macropad на базе Arduino Micro. Программа предоставляет удобный GUI-интерфейс для настройки 6 кнопок в 3 профилях (Game, Work, Art) с возможностью выполнения различных макросов, запуска программ, открытия сайтов, ввода текста и отправки горячих клавиш.

Устройство подключается к компьютеру через USB и синхронизируется с ПО, передавая информацию о нажатиях кнопок и получая текущие параметры профилей.

## ⚡ Функциональные возможности

### Основные функции:
- **Запуск программ и приложений** — указание путей к .exe файлам
- **Открытие веб-сайтов** — автоматическое открытие URL в браузере
- **Выполнение скриптов** — поддержка .bat, .cmd, .ps1 и .py файлов
- **Отправка горячих клавиш** — симуляция нажатий клавиш (например, Ctrl+Shift+Esc)
- **Ввод текста** — автоматический ввод заранее заданного текста
- **Настройка задержек** — добавление пауз между операциями (в секундах)
- **Мультипрофильность** — переключение между тремя профилями (Game, Work, Art)
- **Сохранение конфигурации** — автоматическое сохранение настроек в файл scenarios.json
- **Синхронизация с устройством** — двусторонняя связь с Arduino Micro

### Технические возможности:
- Постоянное соединение с Arduino через Serial порт
- Автоматическое восстановление соединения при отключении устройства
- Фоновый режим работы с треем (сворачивание в системный трей)
- Многопоточная архитектура для ненавязчивой работы

## 🛠 Установка и настройка

### Системные требования:
- **ОС**: Windows (требуется для `os.startfile` и PowerShell-скриптов)
- **Python**: 3.7+
- **Arduino Micro** или совместимое устройство

### Установка зависимостей:

```bash
pip install customtkinter pyserial keyboard pillow pystray
```

### Запуск программы:

```bash
python pad.py
```

### Первичная настройка:

1. Запустите программу `pad.py`
2. Подключите Arduino Micro к компьютеру
3. В выпадающем списке "УСТРОЙСТВО" выберите соответствующий COM-порт
4. Нажмите кнопку "ПОДКЛЮЧИТЬ"
5. Программа автоматически обнаружит устройство и выполнит синхронизацию (около 2.5 секунд)

## 🖥 Интерфейс программы

### Основные элементы:

#### 1. Панель управления портом
```
┌─────────────────────────────────────────────────────────┐
│ УСТРОЙСТВО: [COM3 ▼]             [ПОДКЛЮЧИТЬ]         │
└─────────────────────────────────────────────────────────┘
```
- **COM-порт**: Выбор активного порта для подключения Arduino
- **Статус**: Отображает текущее состояние соединения
  - 🟡 "Синхронизация с COMX..." — подключение
  - 🟢 "СИНХРОНИЗИРОВАНО. СИСТЕМА ГОТОВА." — успешно
  - 🔴 "Соединение прервано..." — ошибка

#### 2. Навигация профилей
```
┌─────────────────────────────────────────────────────────┐
│ [Game ▼]                     [Кнопка 1 ▼]            │
└─────────────────────────────────────────────────────────┘
```
- **Профиль**: Выбор активного профиля (Game/Work/Art)
- **Кнопка**: Выбор конфигурируемой кнопки (1-6)

#### 3. Панель настройки макроса

**Поля ввода (в порядке приоритета выполнения):**

| № | Поле | Описание |
|---|------|----------|
| 1 | **СКРИПТ** | Путь к файлу .bat, .ps1, .py |
| 2 | **ПРОГРАММЫ** | Список путей к .exe файлам (по одной на строку) |
| 3 | **САЙТЫ / ССЫЛКИ** | URL-адреса для открытия (по одной на строку) |
| 4 | **ХОТКЕЙ** | Сочетание клавиш (например, `ctrl+shift+esc`) |
| 5 | **НАПЕЧАТАТЬ ТЕКСТ** | Текст для автоматического ввода |
| 6 | **ЗАДЕРЖКА** | Пауза перед следующим действием (в секундах) |

#### 4. Кнопки действий
```
┌─────────────────────────────────────────────────────────┐
│ [СОХРАНИТЬ]                      [СБРОСИТЬ]          │
└─────────────────────────────────────────────────────────┘
```
- **СОХРАНИТЬ** — сохраняет текущую конфигурацию кнопки
- **СБРОСИТЬ** — очищает все настройки для текущей кнопки

## ⚙️ Конфигурация макросов

### Порядок выполнения действий:

1. Сначала выполняются все указанные пути к программам (`open`)
2. Затем открываются сайты (`sites`)
3. После этого выполняется скрипт (`script`)
4. Отправляется горячая клавиша (`hotkey`)
5. Вводится текст (`text`)
6. Ожидается задержка (`delay`) перед завершением

### Примеры настройки:

#### **Профиль "Game" - Кнопка 1**
- **open**: `C:\Games\Steam\steam.exe`
- **sites**: `https://discord.com/app`
- **Результат**: Открытие Steam и Discord

#### **Профиль "Work" - Кнопка 2**
- **script**: `C:\Scripts\backup.ps1`
- **delay**: `5`
- **Результат**: Запуск PowerShell-скрипта резервного копирования с паузой

#### **Профиль "Work" - Кнопка 6**
- **script**: `C:\Users\arsen\PycharmProjects\Programms\lock.bat`
- **Результат**: Блокировка компьютера

## 🔧 Arduino Micro — Подробное руководство

### Аппаратная конфигурация:

```
Arduino Micro (ATmega32U4)
--------------------------
Пины кнопок:
  D9  -> Кнопка 1 (KEY_1)
  D8  -> Кнопка 2 (KEY_2)
  D5  -> Кнопка 3 (KEY_3)
  D6  -> Кнопка 4 (KEY_4)
  D4  -> Кнопка 5 (KEY_5)
  D7  -> Кнопка 6 (KEY_6)
  A1  -> Потенциометр (громкость)

OLED-дисплей (I2C):
  SDA -> A4
  SCL -> A5
  VCC -> 5V
  GND -> GND
```

### Принцип работы:

1. **Короткое нажатие** (<1 сек): Отправляет `ACTION:<профиль>:<кнопка>` в Python
2. **Длинное нажатие** (≥1 сек) на Кнопку 1: Меняет профиль на аппаратном уровне
3. **Потенциометр**: Автоматически управляет системной громкостью через Consumer HID

### Порядок отправки данных:

```
Arduino -> Python:
  ACTION:0:1     # Нажатие кнопки 1 в профиле 0
  REQ:1          # Запрос данных о профиле 1

Python -> Arduino:
  MAX:3          # Количество профилей
  NAME:Game      # Имя текущего профиля
```

### Требуемые библиотеки для Arduino:

1. **HID-Project** — для эмуляции мультимедийных клавиш (громкость)
2. **Wire** — для I2C связи с OLED
3. **Adafruit_GFX** — графическая библиотека для OLED
4. **Adafruit_SSD1306** — драйвер OLED дисплея

Установка через Library Manager в Arduino IDE:
- Search: "HID-Project" by NicoHood
- Search: "Adafruit SSD1306"
- Search: "Adafruit GFX Library"

## 📄 Формат файла конфигурации

### scenarios.json

Конфигурация сохраняется в файл `scenarios.json` в корневой папке программы.

```json
{
    "port": "COM3",
    "profiles": [
        {
            "name": "Game",
            "buttons": [
                {
                    "open": [
                        "C:\\Games\\Steam\\steam.exe"
                    ],
                    "sites": [
                        "https://discord.com/app"
                    ],
                    "delay": "0.5"
                },
                {},
                {},
                {},
                {},
                {}
            ]
        },
        {
            "name": "Work",
            "buttons": [
                {
                    "script": "C:\\Scripts\\backup.ps1",
                    "delay": "5"
                },
                {},
                {},
                {},
                {},
                {}
            ]
        },
        {
            "name": "Art",
            "buttons": [
                {
                    "hotkey": "ctrl+shift+s"
                },
                {},
                {},
                {},
                {},
                {}
            ]
        }
    ]
}
```

### Структура:

- **port**: COM-порт для подключения
- **profiles**: массив из 3 профилей
  - **name**: имя профиля (Game, Work, Social)
  - **buttons**: массив из 6 кнопок
    - **open**: массив путей к программам
    - **sites**: массив URL-адресов
    - **script**: путь к скрипту
    - **hotkey**: сочетание клавиш
    - **text**: текст для ввода
    - **delay**: задержка в секундах (строка)
    - **close**: название процесса для закрытия

## 🔧 Arduino — Подключение и прошивка

### Шаг 1: Подключение компонентов

1. **Кнопки (6 шт)**:
   - Подключите каждую кнопку между пином D9, D8, D5, D6, D4, D7 и GND
   - Не обязательно использовать резисторы — в коде включено `INPUT_PULLUP`

2. **OLED дисплей (128x64)**:
   - Подключите по I2C: SDA к A4, SCL к A5
   - Питание: VCC (5V), GND

3. **Потенциометр**:
   - Подключите к A1 (центральный пин)
   - Остальные пины к 5V и GND

### Шаг 2: Установка библиотек

В Arduino IDE:
1. Sketch → Include Library → Manage Libraries
2. Найдите и установите:
   - "HID-Project" by NicoHood
   - "Adafruit SSD1306"
   - "Adafruit GFX Library"

### Шаг 3: Загрузка прошивки

1. Откройте файл `pad.ino` в Arduino IDE
2. Выберите устройство: **Tools → Board → Arduino Micro**
3. Выберите COM-порт: **Tools → Port → COMX**
4. Нажмите **Upload**

### Шаг 4: Настройка ПО

1. Запустите `pad.py`
2. В списке портов выберите тот же COM-порт, что использовался в Arduino IDE
3. Нажмите "ПОДКЛЮЧИТЬ"
4. Дождитесь сообщения "СИНХРОНИЗИРОВАНО. СИСТЕМА ГОТОВА."

## 📝 Примечания по безопасному использованию

1. **Блокировка системы**:
   - Файл `lock.bat` должен содержать команду `rundll32.exe user32.dll,LockWorkStation`
   - Будьте осторожны при использовании этой кнопки

2. **Блокировка сайтов**:
   - Скрипт `zapret-discord-youtube` изменяет системные настройки
   - Восстановите настройки после использования

3. **Привилегии**:
   - Некоторые скрипты могут требовать запуска от имени администратора

## 📝 Дополнительная информация

### Горячие клавиши (keyboard library)

Поддерживаемые комбинации:
- `ctrl+c`, `ctrl+v`, `ctrl+a`
- `shift+tab`, `alt+f4`
- `win+d`, `win+e`
- `ctrl+shift+esc`

### Особенности PowerShell-скриптов:

Для запуска PowerShell-скриптов используется параметр `-ExecutionPolicy Bypass`, что позволяет выполнять подписанные и неподписанные скрипты без изменения системной политики.

### Сворачивание в трей:

При закрытии окна (нажатии на "крестик") программа не завершает работу, а сворачивается в системный трей с возможностью:
- Развернуть окно настроек
- Полностью выйти из приложения

### Очередь задач:

Все макросы выполняются в фоновом потоке (`scenario_worker`), который обрабатывает задачи из очереди (`action_queue`). Это предотвращает блокировку интерфейса при длительных операциях.

## 🔧 Устранение неполадок

### Проблема: Устройство не подключается

**Решение:**
1. Проверьте, что Arduino Micro определяется в диспетчере устройств
2. Убедитесь, что выбран правильный COM-порт
3. Убедитесь, что Arduino отправляет данные со скоростью 9600 бод
4. Проверьте соединение проводов

### Проблема: Кнопки не работают

**Решение:**
1. Проверьте, что в конфигурации есть действия для кнопки
2. Убедитесь, что Arduino отправляет команды в формате `ACTION:<профиль>:<кнопка>`
3. Проверьте статус соединения в интерфейсе программы
